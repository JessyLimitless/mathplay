## 엔트로피 최소화로서의 프롬프팅

제2장 전체를 관통해온 하나의 원리가 있다.

**수렴(convergence).**

임베딩(2-1)이 의미를 좌표로 고정하고, 어텐션(2-2)이 관련 정보를 선택적으로 추출하며, 엔트로피(2-3)가 불확실성을 측정하고, RAG(2-4)가 외부 근거로 생성을 앵커링하며, 컨텍스트 윈도우(2-5)가 처리 가능한 정보의 양을 규정한다.

이 모든 것을 아는 조종사에게, 프롬프팅은 더 이상 "AI에게 말 걸기"가 아니다. 그것은 **확률 공간을 체계적으로 압축하는 정보 공학**이다.

## 퓨샷(Few-shot)이 작동하는 수학적 이유

2020년, GPT-3의 가장 놀라운 발견 중 하나는 **퓨샷 러닝(few-shot learning)**이었다. 프롬프트에 몇 개의 예시를 보여주기만 하면, 명시적 학습 없이 패턴을 파악하여 새로운 입력에 적용한다.

```
영어를 한국어로 번역하세요.

영어: Hello → 한국어: 안녕하세요
영어: Thank you → 한국어: 감사합니다
영어: Good morning → 한국어: 
```

모델은 "좋은 아침입니다"를 생성한다. 번역 학습을 한 적이 없는데도.

이것이 작동하는 수학적 이유를 2-3절의 정보이론 프레임으로 설명할 수 있다.

**예시가 없을 때:** "영어를 한국어로 번역하세요"라는 지시만으로는, 모델이 해석할 수 있는 "번역"의 형태가 다양하다. 직역인가 의역인가? 존댓말인가 반말인가? 어떤 문체인가? 출력 공간의 엔트로피가 높다.

**예시가 주어지면:** 두 개의 예시가 강력한 제약 조건으로 작동한다. "Hello → 안녕하세요"에서 모델은 존댓말 스타일을 추론한다. "Thank you → 감사합니다"에서 이 추론이 확증된다. **예시는 사전 확률(prior)을 업데이트하는 베이지안 증거로 기능한다.**

베이즈 정리의 언어로:

> **P(스타일 | 예시들) ∝ P(예시들 | 스타일) · P(스타일)**

예시들이 특정 스타일(존댓말, 직역)과 일관될수록, 사후 확률 P(스타일|예시들)이 그 스타일에 집중된다. 결과적으로 출력 공간의 엔트로피가 급격히 감소한다.

**퓨샷은 통계적 프라이밍(statistical priming)**이다. 몇 개의 예시가 모델의 조건부 확률 분포를 특정 패턴으로 "밀어넣는" 것. 그리고 이 밀어넣기의 효과는 예시의 수보다 **예시의 일관성과 대표성**에 더 크게 의존한다.

## 정보 밀도를 극대화하는 프롬프트 설계 5원칙

2-3절에서 도출한 정보이론적 원칙과 2-5절의 컨텍스트 한계를 결합하면, 실전적 프롬프트 설계 원칙이 도출된다.

### 원칙 1: 역할(Role) — 잠재 공간의 좌표를 지정하라

```
당신은 10년 경력의 데이터 사이언티스트입니다.
```

이 한 문장이 모델의 출력 분포를 "데이터 사이언스 전문가"의 텍스트 패턴 영역으로 이동시킨다. 3장에서 상세히 다루겠지만, 이것은 잠재 공간에서의 사전 확률(prior) 이동이며, 2-3절의 관점에서는 출력 공간의 엔트로피를 **주제 차원에서** 대폭 감소시키는 효과를 갖는다.

### 원칙 2: 맥락(Context) — 배경 정보로 탐색 공간을 한정하라

```
우리 회사는 B2B SaaS 기업이며, 최근 이탈률이 15%에서 22%로 증가했습니다.
MAU는 50,000이고, 주요 고객층은 중소기업 IT 부서입니다.
```

구체적 수치와 상황이 출력 공간을 "일반적 조언"에서 "이 회사에 맞는 조언"으로 압축한다. 정보 이득이 극대. 각 데이터 포인트가 독립적 차원에서 엔트로피를 줄인다(2-3절, 원칙 2).

### 원칙 3: 과제(Task) — 정확히 무엇을 원하는지 명시하라

```
이탈률 증가의 원인을 3가지로 분석하고,
각 원인에 대한 대응 전략을 제시하세요.
```

"분석해줘"보다 "3가지 원인과 각각의 대응 전략"이 정보 밀도가 높다. 출력의 구조(3가지 × 원인 + 전략)가 명시되므로, 모델은 이 구조 안에서 최적의 내용을 채우는 데 집중할 수 있다.

### 원칙 4: 형식(Format) — 출력의 형태를 규정하라

```
마크다운 표 형식으로,
각 원인의 심각도를 상/중/하로 표시하고,
대응 전략의 예상 소요 기간을 포함하세요.
```

형식 지정은 출력 공간의 **구조적 차원**에서 엔트로피를 줄인다. 내용은 모델에게 맡기되, 그릇의 형태는 인간이 결정한다.

### 원칙 5: 제약(Constraints) — 하지 말아야 할 것을 명시하라

```
일반론적 조언은 제외하세요.
"고객 만족도를 높이세요" 같은 뻔한 제안은 불필요합니다.
우리 상황에 특화된 구체적 액션 아이템만 제시하세요.
```

긍정적 제약(해야 할 것)만큼이나 부정적 제약(하지 말아야 할 것)이 중요하다. 부정적 제약은 **출력 공간에서 특정 영역을 명시적으로 차단**하여, 모델이 "안전한 일반론"으로 도피하는 것을 방지한다.

## 제2장을 마치며: 스무고개의 교훈

여섯 개의 절에 걸쳐, 우리는 수렴의 논리를 해부했다.

- 임베딩이 단어를 벡터 공간의 좌표로 변환한다(2-1).
- 어텐션이 모든 토큰 사이의 관련성을 계산하여 유의미한 정보를 선택적으로 추출한다(2-2).
- 엔트로피와 정보 이득이 프롬프트의 각 조건이 출력 공간을 얼마나 좁히는지를 정량화한다(2-3).
- RAG가 파라메트릭 지식의 한계를 외부 검색으로 보완한다(2-4).
- 컨텍스트 윈도우가 처리 가능한 정보의 물리적 한계를 규정한다(2-5).
- 압축적 프롬프팅이 이 모든 원리를 실전에서 활용하는 조종사의 기술이다(2-6).

이 모든 것은 **"수렴"**의 논리였다. 무한한 가능성에서 하나의 답으로 좁혀가는 과정. 임베딩이 의미를 고정하고, 어텐션이 관련성을 선별하며, 프롬프트가 출력 공간을 압축하고, RAG가 사실적 근거에 앵커링한다.

그런데 이 수렴을 **누가** 주도하는가?

기차놀이(제1장)에서 기차는 선로를 따라 달렸다. 스무고개(제2장)에서 질문이 가능성을 좁혔다. 그러나 두 경우 모두, **방향을 설정하는 주체**의 정체에 대해서는 깊이 다루지 않았다.

"너는 수학 교수다"라는 한 문장이 모델의 출력을 극적으로 바꾸는 현상. "단계별로 생각해봐"라는 지시가 추론 능력을 비약적으로 끌어올리는 현상. 이것들은 단순히 "프롬프트 트릭"이 아니다. 잠재 공간에서의 사전 확률 이동이며, 메타인지의 외부 유도이며, **정체성의 조작이 출력을 바꾸는** 현상이다.

이 현상의 수학적 구조와 철학적 함의를 다루는 것이 제3장의 과제다.
